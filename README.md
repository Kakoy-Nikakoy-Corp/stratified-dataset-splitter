# 🐆 Stratified Dataset Splitter

Утилита для стратифицированного разделения датасета, собранного в рамках проекта по обучению модели детекции снежного барса в дикой природе. Обеспечивает воспроизводимое и статистически сбалансированное разбиение данных на обучающую, валидационную и тестовую выборки.

## 🧠 Алгоритм разделения

Жадный алгоритм `greedy_loc_split` последовательно распределяет данные по фолдам с учётом нескольких критериев:

- **📍 Стратификация по локациям**  
  Все изображения с одной камеры/локации (определяется по родительской директории) попадают строго в один фолд. Это исключает `data leakage` между сплитами и сохраняет пространственную независимость выборок.

- **⚖️ Баланс объектов и классов между фолдами**  
  Алгоритм минимизирует отклонение от заданных пропорций (по умолчанию `70/15/15`) сразу по двум метрикам:
  1. Общее количество изображений
  2. Количество размеченных объектов (bounding boxes) целевого класса  
  Это гарантирует, что редкие или плотные локации не исказят распределение класса `snow_leopard` в train/val/test.

- **🎛️ Взвешенная жадная оптимизация (`gain`)**  
  Функция `gain` оценивает, насколько добавление локации в фолд приближает его параметры к целевым значениям. Локации сортируются по убыванию числа позитивных примеров, после чего одна за другой помещаются в те фолды, чей `gain` при добавлении данной группы объектов будет максимален. Параметр `GAIN_WEIGHTS` позволяет гибко задавать приоритеты (по умолчанию баланс размеченных объектов учитывается в 3 раза сильнее, чем баланс количества кадров).

- **🛡️ Ограничение доминирования**  
  Встроенное правило пропускает фолд, если размер локации превышает `25%` от целевого объёма фолда. Это предотвращает ситуацию, когда одна камера формирует большую часть сплита, сохраняя репрезентативность данных.

## 🔄 Конвертация и подготовка данных

Помимо алгоритма разделения, утилита выполняет автоматическую конвертацию датасета из формата экспорта **Label Studio (`JSON_MIN`)** в формат, полностью совместимый с **YOLO**:

- **📐 Приведение координат к YOLO-формату**  
  Аннотации преобразуются из процентного формата Label Studio в нормализованные координаты YOLO `[class_id, x_center, y_center, width, height]`, где все значения находятся в диапазоне `[0, 1]` относительно размеров изображения.
- **🔗 Нормализация путей и хеширование**  
  Базовые URL изображений автоматически заменяются согласно конфигурации (`OLD_URL` → `NEW_URL`). Для избежания конфликтов имён и защиты от спецсимволов файлы именуются с использованием SHA-1 хеша от исходного пути.
- **📏 Автоматическое определение размеров**  
  Если для изображения отсутствуют метки, утилита загружает его и автоматически определяет `width` и `height`, сохраняя целостность структуры датасета и корректность нормализации.
- **📦 Формат NDJSON**  
  Результат сохраняется в `dataset.ndjson`: первая строка содержит метаданные датасета (`YOLO_CONF`), а каждая последующая строка — JSON-объект с путём, размерами, размеченными боксами и назначенным сплитом (`train`/`val`/`test`).

## ⚙️ Установка и запуск

Проект использует `uv` для управления зависимостями и окружением.

```bash
# Установка зависимостей
uv sync

# Запуск скрипта
uv run main.py
```

## 📂 Входные и выходные данные

- **Вход:** `./data/label-studio-export.json` — экспорт из Label Studio с размеченными bounding boxes.
- **Выход:** `./data/dataset.ndjson` — NDJSON-файл, где первая строка содержит конфигурацию датасета в формате YOLO, а каждая последующая строка — описание изображения с полем `"split": "train" | "val" | "test"`.

## 📝 Конфигурация (`config.py`)

| Параметр | Описание |
|----------|----------|
| `SPLIT_RATIO` | Целевые пропорции сплитов `[train, val, test]` |
| `GAIN_WEIGHTS` | Веса для функции оптимизации `[вес_изображений, вес_объектов]` |
| `YOLO_CONF` | Заголовок датасета (имя, классы, задача) |
| `OLD_URL` / `NEW_URL` | Замена базового пути к изображениям при экспорте |
